# מדריך גנרי: מיצוי מידע מדוחות סביבתיים והכנתו לשליפה

## למי המדריך מיועד

כל מי שמתחיל עבודה על **אזור חדש** — אזור תעשייה, מתחם מפעלים, תחנת דלק, מחסן כימיקלים, או כל אתר שבו מתקיים ניטור מי-תהום. האזור יכול להיות מורכב (עשרות מפעלים ומאות קידוחים) או פשוט (מתקן בודד ושני קידוחים). התהליך זהה — רק היקף המקורות משתנה.

---

## מה אנחנו מנסים להשיג

בסוף התהליך, נקבל **מאגר מידע מובנה** שמאפשר שני דברים:

1. **מענה לשאלות מוגדרות מראש** — "מהם מוקדי הזיהום?", "אילו מגמות מסוכנות?", "מי המזהם הסביר?"
2. **מענה לשאלות חופשיות** — "מה קרה בקידוח X ב-2015?", "האם יש קשר בין מפעל Y לזיהום Z?"

שתי היכולות נובעות מאותו מבנה — JSON מובנה עם שדות חיפוש ברורים ואטריבוציית מקור מלאה.

## שלב 0: איסוף מקורות

### מה לאסוף

לכל אזור, יש לאסוף את כל המסמכים הרלוונטיים שקיימים. הרשימה הבאה מסודרת לפי חשיבות יורדת:

| עדיפות | סוג מסמך | דוגמאות | מה מחפשים בתוכו |
|---------|----------|---------|-------------------|
| **חובה** | דוח ניטור שוטף | דוח שנתי של הרשות, דוח רבעוני של יועץ | ריכוזים, קידוחים, מגמות, חריגות |
| **חובה** | סקר היסטורי | דוח תה"ל, סקר אקולוג, סקר קרקע מזוהמת | רקע, מפעלים שפעלו, זיהום עבר |
| **חשוב** | דוח שיקום / תיקון | דוח פעולה מתקנת, תוכנית שיקום | מה תוקן, מה נשאר, האם הצליח |
| **חשוב** | מפות ושרטוטים | מפת קידוחים, חתך גיאולוגי, מפת פלומה | מיקומים, כיוון זרימה, היקף הזיהום |
| **משלים** | רישיון / היתר | רישיון עסק, היתר הזרמה, דוח PRTR | מה מותר להזרים, מה באמת מוזרם |
| **משלים** | חקירה / אכיפה | צו משרד הגנ"ס, דוח פיקוח | הפרות, תאריכים, סנקציות |

### היכן לחפש

```
ערוצי חיפוש מומלצים:

1. רשות המים         — דוחות ניטור שנתיים, מסד "מקורות מים"

5. רשות מקומית       — ועדת תכנון, בקשות לרישיון, מסמכי תב"ע
6. חיפוש Web         — PRTR ישראל, B144, חדשות מקומיות, פורומים שכונתיים
```

**כלל ברזל**: שם הקובץ חייב לשקף את תוכנו. לא `document1.pdf` אלא `survey_soil_contamination_tahal_2007_zone_name.pdf`.

## שלב 1: חילוץ טקסט מ-PDF

### מטרה

להפוך PDF (שהוא בעצם תמונת עמוד) לטקסט שמכונה יכולה לקרוא ולנתח. בעברית, זה מורכב יותר בגלל כיוון הכתיבה (RTL) ותווים מיוחדים.

### כלי העבודה

**`pdftotext`** (חלק מחבילת Poppler) — כלי שורת פקודה שמחלץ טקסט מ-PDF.

```bash
# התקנה (Linux)
apt-get install poppler-utils

# שימוש בסיסי
pdftotext -layout input.pdf output.txt
```

**דגל `-layout`** — קריטי. הוא שומר על מבנה הטבלאות והעמודות כפי שהם מופיעים בעמוד. בלעדיו, טבלאות נהרסות והופכות לערימת מילים.

### ביצוע אוטומטי

```bash
python scripts/extract_zone_pdfs.py --zone <zone_id>
```

**מה הסקריפט עושה, צעד אחר צעד**:

1. **סריקה**: מוצא את כל קבצי ה-PDF בספריית `<Zone>/data/external/`
2. **בדיקת כפילות**: קורא את `_pdf_index.json` (אם קיים) — האם ה-PDF הזה כבר חולץ?
3. **חילוץ**: לכל PDF חדש:
   - מריץ `pdfinfo` לקבל מספר עמודים
   - מריץ `pdftotext -layout` לחלץ טקסט
   - שומר ב-`_raw_text/<שם_קובץ>.txt`
4. **רישום**: מעדכן את `_pdf_index.json` עם מטא-דאטה לכל PDF

### תוצרים

```
<Zone>/data/external/
├── _raw_text/
│   ├── report_monitoring_2023.txt           # טקסט מחולץ
│   ├── survey_historical_tahal_2007.txt
│   └── ...
└── _pdf_index.json                           # אינדקס כל ה-PDFs
```

### אינדקס: `_pdf_index.json`

האינדקס הוא ה-"ספר כתובות" של כל ה-PDFs. הוא מאפשר לדעת מה כבר עובד, מה כשל, ומה חסר.

```json
{
  "zone_id": "zone_name",
  "extraction_date_utc": "2026-06-14T12:00:00Z",
  "pdf_count": 4,
  "pdfs": [
    {
      "filename": "report_monitoring_2023.pdf",
      "page_count": 87,
      "size_bytes": 15200000,
      "char_count_extracted": 142000,
      "extraction_ok": true,
      "extracted_at": "2026-06-14T12:00:05Z"
    },
    {
      "filename": "survey_historical_tahal_2007.pdf",
      "page_count": 210,
      "size_bytes": 45000000,
      "char_count_extracted": 0,
      "extraction_ok": false,
      "error": "scanned image — no embedded text layer"
    }
  ]
}
```

### בעיות שכיחות ופתרונות

| בעיה | סימפטום | פתרון |
|------|---------|-------|
| **PDF סרוק** (תמונה בלבד) | `extraction_ok: false`, `char_count: 0` | OCR ידני (ABBYY, Google Vision), או חילוץ ידני של טבלאות |
| **עברית מעורבלת** | אותיות הפוכות, מילים שבורות | בדיקה ידנית של עמודים קריטיים; שימוש ב-`-raw` במקום `-layout` |
| **טבלאות מורכבות** | עמודות מתערבבות, merge-cells | חילוץ ידני לקובץ CSV נפרד; תיעוד עמוד מקור |
| **PDF מוצפן** | שגיאת הרשאה | בקשת גרסה לא מוצפנת מהרשות |

## שלב 2: עיבוד AI — הפיכת טקסט לנתונים מובנים

### מטרה

טקסט גולמי הוא חסר ערך אנליטי. מה שאנחנו צריכים הוא **מידע מובנה**: אילו קידוחים מוזכרים, אילו מזהמים נמצאו, מתי, בכמה, ומה המשמעות. שלב זה ממיר טקסט חופשי ל-JSON שניתן לשלוף ממנו.

### עקרון הפעולה

לכל PDF שחולץ בשלב 1, **סוכן AI** קורא את הטקסט המחולץ ומוציא ממנו מידע מובנה בפורמט קבוע. הסוכן מקבל הנחיות מקצועיות (persona של הידרוגיאולוג) ומענה בהתאם.

### מה הסוכן מחפש

הסוכן סורק את הטקסט ומחלץ **7 קטגוריות מידע**:

#### 1. קידוחים שמוזכרים

כל אזכור של שם קידוח, ציון מיקום, עומק, סוג (ניטור/הפקה), שכבת אקוויפר.

```json
{
  "name_he": "נת רעננה 12",
  "name_en": "NT_Raanana_12",
  "coordinates_itm": {"east": 189450, "north": 685230},
  "depth_m": 52.0,
  "classification": "monitoring",
  "aquifer_layer": "אקוויפר חוף — תת-שכבה B",
  "source_page": 34
}
```

**למה זה חשוב**: מזהה את הגיאוגרפיה. קידוח שמוזכר בדוח ב-2007 ולא בדוח 2023  — אולי הופסק ניטור  — פער.

#### 2. ממצאי זיהום

כל ציון של מזהם, ריכוז, תאריך, קידוח, ויחס לתקן. זהו לב הנתונים.

```json
{
  "parameter": "Trichloroethylene",
  "family": "CVOC",
  "concentration_ug_l": 85.3,
  "date": "2007-06-15",
  "well": "נת רעננה 12",
  "dws_value": 7.5,
  "percent_of_standard": 1137,
  "severity_qualitative": "חריגה משמעותית",
  "interpretation_he": "ריכוז גבוה פי 11 מהתקן, עלייה ביחס לדגימה קודמת",
  "source_page": 45
}
```

**למה זה חשוב**: מאפשר השוואה בין-תקופתית. TCE ב-2007 = 85 µg/L, ב-2023 = 12 µg/L  — ירידה  — אולי שיקום עובד.

#### 3. מקורות זיהום חשודים

כל אזכור של מפעל, מתקן, תחנת דלק, מחסן, או כל מקור אנושי שעלול לגרום לזיהום.

```json
{
  "name_he": "מפעל ציפוי מתכות א.ב. בע\"מ",
  "address_he": "רחוב התעשייה 15, אזור התעשייה",
  "suspected_contaminants": ["Chromium", "Nickel", "Cyanide"],
  "confidence": "HIGH",
  "evidence_he": "קרבה 150 מ' לקידוח, חריגות כרום באותו קידוח מ-2005, ידוע שהשתמשו בציפוי גלווני",
  "operating_years": "1985-2015",
  "current_status": "סגור — נמצא בתהליך שיקום",
  "source_page": 67
}
```

**רמת ביטחון**:

| רמה | קריטריונים |
|-----|-----------|
| **HIGH** | כתובת מאומתת + מזהם תואם בקידוח קרוב (<500 מ') + שנות פעילות חופפות |
| **MEDIUM** | 2 מתוך 3 הקריטריונים |
| **LOW** | קריטריון יחיד, או אזכור כללי ללא ראייה ישירה |

#### 4. הקשר הידרוגיאולוגי

כיוון זרימה, סוג אקוויפר, עונתיות, קשרים בין שכבות.

```json
{
  "flow_direction": "מזרח-מערב, שיפוע 0.3%",
  "aquifer_type": "אקוויפר חוף, חול אאולייני, מוליכות 15-25 מ'/יום",
  "seasonal_variation": "עליית מפלס בחורף (ינואר-מרץ)  — דילול",
  "cross_layer_connection": "קשר אפשרי בין שכבות B ל-C דרך חלון גיאולוגי באזור הדרומי",
  "source_page": 23
}
```

**למה זה חשוב**: בלי הידרוגיאולוגיה, אי אפשר להסביר למה קידוח מסוים מזוהם ואחר לא. כיוון הזרימה מחבר בין מקור לקידוח.

#### 5. מגמות

כל תיאור של שינוי בריכוז לאורך זמן — עלייה, ירידה, יציבות, תנודות עונתיות.

```json
{
  "text_he": "TCE בקידוח נת רעננה 12 ירד מ-85 ל-12 µg/L בין 2007 ל-2023 — ירידה מתמדת, ככל הנראה בעקבות הפסקת שימוש ב-TCE על ידי המפעל",
  "parameter": "TCE",
  "well": "נת רעננה 12",
  "direction": "decreasing",
  "period": "2007-2023",
  "source_page": 51
}
```

#### 6. ציטוטים מפתח

משפטים חשובים שמגלמים שיפוט מקצועי, מסקנה, או עובדה קריטית — בניסוח המקורי מהדוח.

```json
{
  "text_he": "לאור ריכוזי הבנזן הגבוהים ותבנית ההתפשטות, קיימת סבירות גבוהה שמקור הזיהום הוא מכלי הדלק התת-קרקעיים של תחנת הדלק ברחוב הרצל",
  "significance": "attribution — facility linkage",
  "page": 72
}
```

**למה זה חשוב**: ציטוט ישיר מדוח מומחה הוא ראיה ברמה A — החזקה ביותר. זה לא פרשנות של AI אלא קביעה של הידרוגיאולוג המקורי.

#### 7. המלצות

פעולות שהדוח המקורי ממליץ — הגברת ניטור, חקירה, שיקום, סגירת קידוח.

```json
{
  "text_he": "מומלץ להגביר את תדירות הדיגום בקידוח נת רעננה 12 לאחת לרבעון ולהוסיף בדיקת PFAS",
  "urgency": "high",
  "domain": "monitoring",
  "source_page": 88
}
```

### Schema מלא: `_findings_<tag>.json`

כל PDF מייצר קובץ `_findings_<tag>.json` כאשר `<tag>` הוא מזהה קצר של ה-PDF (לדוגמה: `tahal2007`, `monitoring2023`, `remediation_site_x`).

```json
{
  "source_file": "survey_historical_tahal_2007.pdf",
  "title_he": "סקר אתרי קרקע מזוהמת — אזור תעשייה",
  "year": 2007,
  "author_org_he": "תה\"ל — תכנון המים לישראל",
  "summary_he": "סקר מקיף של 12 אתרים פוטנציאליים באזור התעשייה. זוהו 4 מוקדי זיהום עיקריים: CVOC ממפעלי מתכת, דלק מתחנות, כרום מציפוי גלווני.",
  "boreholes_mentioned": [ ... ],
  "contamination_findings": [ ... ],
  "facilities_suspected": [ ... ],
  "hydrogeology_he": { ... },
  "trends_described_he": [ ... ],
  "key_quotes_he": [ ... ],
  "recommendations_he": [ ... ]
}
```

### סיווג אמינות הראיה

כל פיסת מידע שמחולצת מקבלת סיווג אוטומטי לפי **מערכת A–E**:

| דרגה | קוד | משמעות | כיצד מזהים |
|------|-----|--------|-----------|
| **A** | `raw_report_verified` | אומת ישירות מול הטקסט הגולמי של הדוח | המשתמש קרא את העמוד ואישר |
| **B** | `ai_extracted_with_page_ref` | AI חילץ עם הפניה לעמוד מקור | יש `source_page` ב-JSON |
| **C** | `web_verified_current_activity` | מידע מ-Web שאושר עדכני | אתר חי, PRTR, B144 |
| **D** | `inferred_candidate` | הסקה לוגית — אין ראייה ישירה | קרבה גיאוגרפית, שם רחוב, הקשר |
| **E** | `weak_or_mention_only` | אזכור חלוש, חד-פעמי, לא מגובה | הערת שוליים, רשימה ארוכה |

**כלל שימוש**:
- **A + B** — נכנסים לגוף הדוח כממצא. ניתן לצטט.
- **C** — רקע בלבד. "המפעל פעיל" ≠ "המפעל מזהם".
- **D + E** — נספח או רקע. ייכנסו לגוף רק אם נתוני ניטור (מדידות בפועל) מחזקים.

## שלב 3: איחוד ממצאים

### מטרה

לכל אזור ייתכנו 3, 5, או 15 מסמכי PDF. כל אחד ייצר `_findings_*.json` משלו. שלב 3 מאחד את כולם למסמך יחיד, חד-משמעי, ללא כפילויות, עם אטריבוציית מקור מלאה.

### ביצוע

```bash
python scripts/merge_extracted_findings.py --zone <zone_id>
```

### לוגיקת האיחוד

**קידוחים** — דדוקציה לפי שם:
- קידוח "נת רעננה 12" מופיע ב-3 PDFs  — שמור פעם אחת
- הראשון שמוזכר עם קואורדינטות  — הוא שנשמר
- אם שני מקורות סותרים בקואורדינטות  — WARN (ייבדק ידנית)

**ממצאי זיהום** — שמור הכל:
- אותו קידוח + אותו מזהם + שנים שונות = שני רשומות נפרדות (זהו ערך — נתוני זמן!)
- אותו קידוח + אותו מזהם + אותה שנה מ-2 מקורות = שמור שניהם עם `source_file` שונה

**מקורות חשודים** — דדוקציה לפי שם + ביטחון:
- אם PDF A אומר "מפעל X, HIGH" ו-PDF B אומר "מפעל X, MEDIUM"  — שמור HIGH
- מיזוג ראיות: evidence strings מכל המקורות מתחברים

**מגמות / המלצות / ציטוטים** — שמור הכל:
- דדוקציה של טקסט זהה בדיוק (exact text match)
- טקסט דומה אך לא זהה  — שמור שניהם (ייתכן הבדל מהותי)

## שליפת מידע מהמאגר

### עקרון

`extracted_findings.json` הוא מאגר שטוח (flat JSON) שניתן לשלוף ממנו בשני אופנים:

### א. שאלות מוגדרות מראש (Predefined Queries)

אלו 8 השאלות שהדיאגנוזה המקצועית (Zone Diagnosis) שואלת. כל אחת מהן מיושמת כשליפה ממוקדת מתוך ה-JSON:

| # | שאלה | שדה/שדות לשליפה | לוגיקה |
|---|------|-----------------|--------|
| 1 | **מהם מוקדי הזיהום המרכזיים?** | `contamination_findings[]` | קיבוץ לפי קידוח  — קיבוץ לפי קרבה גיאוגרפית (ITM ±200 מ')  — דירוג לפי max(percent_of_standard) |
| 2 | **אילו קידוחים מובילים כל מוקד?** | `boreholes[]` + `contamination_findings[]` | JOIN על `well`  — סינון לפי אזור גיאוגרפי  — מיון לפי חומרה |
| 3 | **אילו מזהמים מגדירים כל מוקד?** | `contamination_findings[]` | קיבוץ לפי `family` בתוך כל מוקד  — מיון לפי max(percent_of_standard) |
| 4 | **מה השתנה מאז דוחות קודמים?** | `contamination_findings[]` | השוואת ריכוזים באותו קידוח+מזהם בשנים שונות; `trends[]` כמקור משלים |
| 5 | **אילו מגמות מסוכנות?** | `trends[]` + נתוני מגמות חיצוניים (CSVs) | סינון: `direction == "increasing"` + `parameter.family in [CVOC, FUEL, METALS, PFAS]` |
| 6 | **אילו פערי ניטור קריטיים?** | `boreholes[]` + `recommendations[]` | קידוחים שהוזכרו היסטורית אך לא מופיעים בנתונים עדכניים; המלצות ניטור שלא יושמו |
| 7 | **אילו מקורות ראויים לדיון?** | `facilities_suspected[]` | סינון: `confidence in ["HIGH", "MEDIUM"]`  — מיון לפי proximity לקידוחים חורגים |
| 8 | **מה לגוף הדוח ומה לנספח?** | הצלבה של כל השדות | `evidence_class A+B`  — גוף; `D+E`  — נספח; ממצאים ללא גיבוי מנתוני ניטור  — נספח |

**דוגמת שליפה** (שאלה 1 — מוקדי זיהום):

```python
import json

with open("extracted_findings.json") as f:
    data = json.load(f)

# שלב א: קיבוץ ממצאים לפי קידוח
by_well = {}
for finding in data["contamination_findings"]:
    well = finding["well"]
    by_well.setdefault(well, []).append(finding)

# שלב ב: דירוג קידוחים לפי חומרה מקסימלית
ranked = sorted(
    by_well.items(),
    key=lambda x: max(f.get("percent_of_standard", 0) for f in x[1]),
    reverse=True
)

# שלב ג: הצגת 5 הקידוחים החמורים ביותר
for well, findings in ranked[:5]:
    worst = max(findings, key=lambda f: f.get("percent_of_standard", 0))
    print(f"{well}: {worst['parameter']} — {worst['percent_of_standard']}% מהתקן")
```

### ב. שאלות חופשיות (Ad-hoc Queries)

כל שאלה חופשית מתורגמת ל-**חיפוש + סינון + הצגה** על פני שדות ה-JSON. הנה דוגמאות:

**"מה קרה בקידוח X?"**
```python
well_name = "נת רעננה 12"

# כל הממצאים בקידוח זה
findings = [f for f in data["contamination_findings"] if f["well"] == well_name]

# ממוין כרונולוגית
findings.sort(key=lambda f: f.get("date", ""))

# כל המגמות שמזכירות את הקידוח
trends = [t for t in data["trends"] if well_name in t.get("text_he", "")]

# כל הציטוטים שמזכירים את הקידוח
quotes = [q for q in data["key_quotes"] if well_name in q.get("text_he", "")]
```

**"האם יש קשר בין מפעל Y לזיהום Z?"**
```python
facility_name = "מפעל ציפוי מתכות"
contaminant = "Chromium"

# מצא את המפעל
facility = next(
    (f for f in data["facilities_suspected"]
     if facility_name in f.get("name_he", "")),
    None
)

if facility:
    # מזהמים חשודים
    suspects = facility.get("suspected_contaminants", [])
    match = contaminant in suspects

    # חיפוש קידוחים קרובים עם אותו מזהם
    nearby_findings = [
        f for f in data["contamination_findings"]
        if f.get("parameter", "").upper() == contaminant.upper()
    ]

    print(f"מפעל: {facility['name_he']}")
    print(f"ביטחון: {facility['confidence']}")
    print(f"{contaminant} ברשימת חשודים: {'כן' if match else 'לא'}")
    print(f"ממצאי {contaminant} באזור: {len(nearby_findings)}")
```

**"מה עדיין לא נבדק?"**
```python
# קידוחים מדוחות ישנים שלא מופיעים בניטור הנוכחי
historical_wells = {b["name_he"] for b in data["boreholes"]}

# (מפעל measurements_scoped.csv)
import csv
with open("<Zone>/02_data/measurements_scoped.csv") as f:
    current_wells = {row["canonical_well_id"] for row in csv.DictReader(f)}

orphaned = historical_wells - current_wells
print(f"קידוחים היסטוריים ללא ניטור נוכחי: {len(orphaned)}")
for w in sorted(orphaned):
    print(f"  • {w}")
```

### ג. חיפוש טקסט חופשי (Full-text)

לשאלות שלא ניתנות למיפוי ישיר לשדות — חיפוש טקסט חופשי על כל שדות ה-JSON:

```python
import re

query = "בנזן"  # או כל מילה/ביטוי

results = []
for section_name in ["contamination_findings", "trends", "key_quotes", "recommendations"]:
    for item in data.get(section_name, []):
        text = json.dumps(item, ensure_ascii=False)
        if re.search(query, text, re.IGNORECASE):
            results.append({"section": section_name, "item": item})

print(f"נמצאו {len(results)} תוצאות עבור '{query}':")
for r in results[:5]:
    print(f"  [{r['section']}] {json.dumps(r['item'], ensure_ascii=False)[:120]}...")
```

## ממה מורכב מאגר טוב

### Checklist לפני שימוש

לפני שמתחילים לשלוף מ-`extracted_findings.json`, יש לוודא:

- [ ] **כיסוי מקורות**: האם כל ה-PDFs הרלוונטיים עברו חילוץ ועיבוד?
- [ ] **אטריבוציה**: לכל entry יש `source_file` — ניתן לחזור למקור?
- [ ] **דדוקציה**: אין קידוחים כפולים ב-`boreholes[]`?
- [ ] **סבירות תאריכים**: כל הממצאים בטווח הגיוני (1980–2026)?
- [ ] **סבירות ריכוזים**: אין ערכים חריגים (999,999 µg/L) שמעידים על שגיאת חילוץ?
- [ ] **מקורות חשודים**: לכל מפעל יש `confidence` תקין (HIGH/MEDIUM/LOW)?
- [ ] **קואורדינטות**: אם נתונות — בטווח סביר לאזור הגיאוגרפי?
- [ ] **עמוד מקור**: לממצאים קריטיים (percent_of_standard > 100%) יש `source_page`?

### מתי המאגר לא מספיק

המאגר הוא **תמצית ממסמכים**. הוא לא מחליף:

- **נתוני ניטור עדכניים** — מדידות בפועל (`02_data/measurements_scoped.csv`) הם ה-SSOT
- **ניתוח מגמות סטטיסטי** — Mann-Kendall מ-CSVs, לא מטקסט דוח
- **מפות** — קואורדינטות ITM מדויקות מ-Excel, לא מקירוב בטקסט
- **שיפוט מומחה** — המאגר מספק input; ההחלטה היא של ההידרוגיאולוג
