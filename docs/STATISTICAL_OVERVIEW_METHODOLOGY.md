## 1. עקרונות מנחים

### 1.1 הפרדת מסגרות ניטור (קריטי)

קידוחי ניטור באזור תעשייה משרתים שני מטרות שונות שיש להפריד ביניהן בכתיבה:

| מסגרת | אפיון | מה לנטר | היכן ממוקמים |
|--------|--------|----------|----------------|
| **ניטור כללי** | מסגרת בסיסית | CVOC, מתכות, PFAS | מוקדי תעשייה כלליים |
| **ניטור דלק (משלים)** | שימוש נוסף | BTEX, MTBE | תחנות דלק, חניוני אוטובוסים |

**כלל**: הסטטיסטיקה ב**פסקה הראשית** מתבססת על קידוחי ניטור כללי בלבד. ניטור דלק מובא ב**פסקה נפרדת בסוף** כניתוח משלים.

**רציונל**: 
- קידוחי דלק נקדחו ייעודית סמוך למקורות דלק — הם לא נועדו לאפיין את האזור.
- ערבוב יוצר תחושה שגויה של "ניטור חלקי" (X מתוך Y קידוחים נבדקו ל-CVOC).
- מי שבדק TCE בקידוח, גם אם נדגם לדלק — נחשב לקידוח ניטור כללי.

### 1.2 חלון זמן

**5 שנים אחרונות בלבד** (לדוח 2026: 2021-01-01 עד 2026-05-11).

**רציונל**: 
- נתונים ישנים יותר משקפים מצב היסטורי, לא המצב בפועל.
- המספרים בדוח חייבים לשקף **מציאות עכשווית**, לא ממוצעים היסטוריים.

### 1.3 יחידת ספירה: קידוחים, לא מדידות

עבור כל זוג (קידוח, פרמטר), נלקחת **המדידה האחרונה בלבד** בתקופה. הסטטיסטיקה מבוססת על **מספר קידוחים שנמדדו**, לא על סך המדידות.

**רציונל**: למניעת הטיה לקידוחים שנמדדו מספר פעמים.

### קבצי קלט (לכל אזור)

|------|------|------|
| `severity_index_2025_<zone>_param_level.csv` | מדידה אחרונה לכל זוג (קידוח, פרמטר) + אינדקס חומרה | `build_<zone>_borehole_classification.py` |

### עמודות חובה ב-`severity_index_*_param_level.csv`

- `borehole` — מזהה קידוח
- `family` — משפחת זוהמים (`INDUSTRY`/`METALS`/`PFAS`/`FUEL`)
- `param_name` — שם פרמטר
- `date` — תאריך מדידה (yyyy-mm-dd)
- `concentration` — ריכוז [µg/L]
- `drinking_water_standard` — תקן מי שתייה
- `pct_of_standard` — % מהתקן
- `bucket` — אינדקס חומרה (0-8)

## 4. תהליך החישוב

### 4.1 סינון נתונים

```python
df = pd.read_csv('severity_index_2025_<zone>_param_level.csv')
df['date'] = pd.to_datetime(df['date'])
df_recent = df[df['date'] >= '2021-01-01']  # adjust for current year - 5
```

### 4.2 הפרדת קבוצות קידוחים

```python
# קידוחי ניטור כללי = כל קידוח עם מדידה לא-FUEL
general_bh = set(df_recent[df_recent['family'] != 'FUEL']['borehole'].unique())

# קידוחי דלק בלבד = קידוחים עם FUEL ובלי מדידות אחרות
fuel_only_bh = set(df_recent[df_recent['family'] == 'FUEL']['borehole'].unique()) - general_bh

# סה"כ
total_bh = df_recent['borehole'].nunique()
assert total_bh == len(general_bh) + len(fuel_only_bh) + <partial>
```

**הערה**: ייתכנו קידוחים עם ניטור חלקי (למשל מדידה יחידה של 1,4-Dioxane בלבד). יש לבדוק ולהתחשב בכך — למשל לסווגם תחת "53 קידוחי דלק נוספים" או לפרק לפי הרכב.

### 4.3 התפלגות חומרה (לקידוחי ניטור כללי)

```python
# מקסימום אינדקס לכל קידוח על פני פרמטרים לא-FUEL
non_fuel = df_recent[(df_recent['borehole'].isin(general_bh)) & 
                     (df_recent['family'] != 'FUEL')]
max_bucket_per_bh = non_fuel.groupby('borehole')['bucket'].max()

# קיבוץ ל-5 דרגות
def severity_label(b):
    if b == 0: return 'נקי'
    if b <= 3: return 'נמוך'
    if b <= 5: return 'בינוני'
    if b <= 7: return 'גבוה'
    return 'גבוה מאוד'

dist = max_bucket_per_bh.apply(severity_label).value_counts()
```

### 4.4 כיסוי ניטור לכל פרמטר

לכל משפחת CVOC (TCE, PCE, 1,1-DCE, 1,4-Dioxane, Vinyl Chloride, cis/trans-1,2-DCE, Chloroform, 1,2-DCA, Carbon Tetrachloride):

```python
for param in cvoc_params:
    sub = df_recent[df_recent['param_name'] == param]
    n_boreholes = sub['borehole'].nunique()
    crossing = (sub['pct_of_standard'] > 100).sum()
    pct_crossing = 100 * crossing / n_boreholes
    max_conc = sub['concentration'].max()
```

## 5. מבנה הסעיף הסטטיסטי בדוח

### 5.1 רכיבים חובה (סדר קבוע)

1. **פסקה 1 — סקירה כללית**: 
   - מספר קידוחי ניטור כללי (לא סה"כ!)
   - התפלגות 5-דרגות חומרה (אחוזים)
   - חומרה מקסימלית
   
2. **טבלה 1 — התפלגות חומרה** (5 שורות לפי דרגה)

3. **פסקה 2 — מוכלרים תעשייתיים (CVOC)**:
   - **TCE** במרכז: % חוצים תקן, ריכוז מקסימלי, סטטוס IARC Group 1
   - **1,4-Dioxane**: % חוצים, פי כמה מהתקן
   - **Vinyl Chloride**: זיהוי כתוצר פירוק
   
4. **טבלה 2 — כיסוי CVOC** (TCE, 1,1-DCE, 1,4-Dioxane, PCE, VC, cis-1,2-DCE, Chloroform, trans-1,2-DCE, 1,2-DCA, Carbon Tetrachloride)

5. **פסקה 3 — ניטור דלק (משלים, קצר)**:
   - הסבר על מסגרת נפרדת (53 קידוחים ייעודיים)
   - MTBE + Benzene כממצאים עיקריים
   - הדגשה: זיהום נקודתי, לא אזורי (<500 מ' מהמוקד)
   - הפנייה לסעיף מפורט בהמשך הדוח

6. **טבלה 3 — ערכי דלק** (Benzene, MTBE, Toluene, Xylene)

7. **פסקת הערות מתודולוגיות**:
   - הסבר 5 דרגות החומרה
   - הסבר אינדקס 0-8
   - הסתייגות: קידוחים בנקודות חשודות  — לא נציג של האזור

### 5.2 אורך מומלץ

400-450 מילים סה"כ. אם ארוך יותר — סימן שהוצאתם פרטים שצריכים סעיף נפרד.

## 9. שגיאות נפוצות (lessons from חולון)

### 9.1 ערבוב מסגרות

❌ **שגוי**: "מערכת הניטור כוללת 80 קידוחים. רק 27 נבדקו ל-CVOC."

✅ **נכון**: "מערכת הניטור באזה"ת לזיהומים תעשייתיים כוללת 27 קידוחים."

**רציונל**: אזכור 80 ואז סייג ל-27 יוצר תחושת ניטור חלקי. הניטור הוא של 27 קידוחים — קידוחי הדלק הם מסגרת נפרדת.

### 9.2 שימוש במונח "עקבה"

❌ **שגוי**: "סף עקבה — דורש מעקב מתמשך"

✅ **נכון**: "בינוני — בסביבת התקן או חוצה במעט (אינדקס 4-5)"

### 9.3 הצגת אינדקס היסטורי כאקטואלי

❌ **שגוי**: שימוש ב-`severity_bucket` מ-`borehole_classification_all.csv` (מבוסס 2018+).

✅ **נכון**: סינון `date >= 2021-01-01` ולחשב אינדקס מחדש על נתונים עדכניים בלבד.

### 9.4 מספרים מעוגלים

❌ **שגוי**: "כ-30 קידוחים", "~40-50%"

✅ **נכון**: "27 קידוחים", "33% (9/27)"

**רציונל**: דוח מקצועי דורש מספרים מדויקים. אם המספר לא ברור — בדוק את הנתונים.
