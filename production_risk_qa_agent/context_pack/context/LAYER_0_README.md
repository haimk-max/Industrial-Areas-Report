# Layer 0: Core/Synthesized Context — תשובות 90% של השאלות

**כל שאלה שנכנסת למערכת צריכה להתחיל בשכבה זו.**

---

## מה זה Layer 0?

קבצים **עיבודים ומסונתזים** של ממצאים — ללא צורך לחפור בדוחות היסטוריים. זה מכיל:
- **דוח סיכום מלא** (V8) עם כל התוצאות אחרי סינתזה
- **נתונים כמותיים** (CSVs) — severity, trends, monitoring gaps
- **פירוש אומתי** של עדויות (decay chains, source signatures)

## מתי להשתמש בShכבה 0?

✅ **שאלות סטנדרטיות**:
- "מה מצב קידוח X?"
- "אילו קידוחים בסיכון?"
- "מה המגמות הקריטיות?"
- "אילו פערי ניטור?"
- "מה הסיכון לקידוחי הפקה?"

✅ **Type B Q&A** (תשובות קצרות + כרטיסים)

❌ **לא מומלץ** לשכבה 0:
- "אימות עדות היסטורית"
- "השוואת דוחות שונים"
- "מה בדיוק חזר בשנת 2007?"
  → זקוקים לShכבה 1 או 2

## איך לנווט בShכבה זו

1. **לשאלה על קידוח ספציפי** → חפש בתוך `HOLON_REPORT_V8.md` (Ctrl+F) או בCSVs
2. **לשאלה על סיכון כללי** → ראה `zone_diagnosis.md` סעיף "Risk Summary"
3. **לשאלה על מקור** → ראה `forensics_brief.md` או `source_candidates_context.md`
4. **לשאלה על מגמה** → `data/trends_by_well_parameter.csv` + `HOLON_REPORT_V8.md` סעיף "Trends"

## דוגמה: שאלה בShכבה 0

**שאלה**: "מה הסיכון לקידוח מק חולון 12?"

**כיצד למצוא**:
1. חפש "מק חולון 12" בתוך `HOLON_REPORT_V8.md` → קבל סעיף מלא
2. תסתכל ב-`data/severity_by_well_family.csv` → מצא severity scores
3. תסתכל ב-`data/trends_by_well_parameter.csv` (סינון לקידוח זה) → מצא מגמות
4. תסתכל ב-`forensics_brief.md` סעיף decay chains → בדוק קו-הופעה בקידוחי ניטור

**תשובה**: סינתזה מכל 4 → "קידוח זה בסיכון [תיאור] בגלל [עדויות]"

## חוזק + מגבלה של Shכבה 0

✅ **חוזק**:
- עיבוד מלא — כל מה שצריך לרוב השאלות
- סינתזה מאומתת — לא צריך לחפור בדוחות
- מהיר — קריאה + חיפוש בקבצים קטנים

⚠️ **מגבלה**:
- אם צריך אימות עדות היסטורית — עברי לShכבה 1/2
- אם צריך להשוות לדוח מסוים — Shכבה 1/2
