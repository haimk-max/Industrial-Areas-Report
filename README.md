## Running the Pipeline (any zone)

Every script accepts `--zone <id>` and derives all paths from the zone name:

```bash
# Step 1: Excel ingestion (zone-specific Excel auto-detected via config/zone_overrides/<zone>.yaml)
python scripts/parse_excel.py --zone <zone>

# Step 2: Trend analysis (Mann-Kendall with SNR gating)
python scripts/trend_analysis.py --zone <zone>

# Step 3: Forensic attribution
python scripts/forensics_analyzer.py --zone <zone>

# Step 4: Tier 1/2/3 borehole selection (polygon intersection for Tier 2)
python scripts/select_boreholes.py --zone <zone> --list-tiers

# Step 5: Charts (zone-specific for Raanana; data-driven generic for any other zone)
python scripts/generate_charts_v2.py --zone <zone>

# Step 6: Validate report quality (3 automated checks)
python scripts/validate_report.py --report <Zone>/output/<ZONE>_REPORT_V1.md --charts-dir <Zone>/charts_v2

# Regression tests
pytest tests/ -v
```

## Adding a New Zone

See **CLAUDE.md § 8** for the canonical step-by-step procedure. Brief summary:

1. **Upload data**:
   - Excel measurements → `Water Quality Data/`
   - Background PDFs → `<Zone>/data/external/`
   - KMZ polygon → `zone_definitions/` (will be converted to ITM in `zone_polygons.json`)

2. **Configure**:
   - `config/zone_overrides/<zone>.yaml` if Excel column layout differs from Raanana defaults
   - `zone_definitions/zone_polygons.json` — populate ITM polygon coordinates
   - `zone_definitions/tier1_historical_boreholes.json` — historical borehole IDs

3. **Run pipeline**:
   ```bash
   python scripts/parse_excel.py        --zone <X>
   python scripts/trend_analysis.py     --zone <X>
   python scripts/forensics_analyzer.py --zone <X>
   python scripts/generate_charts_v2.py --zone <X>
   ```

4. **Write zone deliverables** (manual, using Raanana as template):
   - Zone summary report
   - Drilling cards per Tier 1 borehole
   - Facility discovery (AI agent)

5. **Submit to expert hydrogeologist** for review.

Requires: Ministry of Environmental Protection coordination per zone.
